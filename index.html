◆ DHT Globe — User Guide

DHT Globe is a browser-based simulator for Distributed Hash Table protocols. Nodes are placed on a 3-D globe; lookups route messages through the network hop by hop. Classical protocols (Kademlia, Geographic) use static routing tables. Neuromorphic protocols learn from traffic — synaptic shortcuts form on heavily-used paths and decay on idle ones, letting the routing table adapt over time. The current state of the art is NX-17: 5–7× faster local routing than Kademlia, near-invariant under 25% churn, and a self-healing distributed pub/sub membership protocol with publisher-prefix addressing, dynamic topic trees, and a bounded replay cache. NH-1 compresses NX-17's ~36 organically-grown rules into five fundamental operations through a unified vitality model — fewer parameters, same AxonManager pub/sub stack, and within ~1–2 percentage points of NX-17 on every delivery metric.

Simulator Integrity

Every benchmark in this simulator is produced under three explicit invariants, each guarded by an automatic check that surfaces violations to the browser console:

  • Bilateral connection cap. Every node enforces connections.size ≤ maxConnections (default 100). After bootstrap, after warmup, and after every churn round, DHT.verifyConnectionCap() walks all live nodes and emits [CAP VIOLATION] as a console error if any node exceeds its cap. When Web Limit is unchecked the cap is Infinity and the check is a no-op.
  • Locality. No optimisation reads another node's routing table directly. Each node's findKClosest runs independently using only its own state — exactly what a real WebRTC client could observe.
  • Bounded RPC. findKClosest simulates real Kademlia FIND_NODE: each peer queried during iterative discovery responds with up to k=20 nodes from its own routing table — never a full routing-table dump. Matches the wire-bounded response in real Kademlia.
Quick Start
  1. Choose a Protocol and set the Node count, then click Init to build the network.
  2. Click Lookup Test to run random key lookups and see hop-count and latency stats.
  3. Click Demo to animate a single lookup — arcs trace each hop on the globe.
  4. Click Train Network to run repeated sessions on any protocol. Classical protocols (Kademlia, Geographic) refine their k-buckets with fresher peer contacts; neuromorphic protocols additionally build synaptic shortcuts and show larger hop-count reductions over time.
  5. Click Benchmark to compare selected protocols side-by-side across selected test conditions.

Drag the globe to rotate · scroll to zoom · click any node to show its routing table connections.

Tests & Modes
ButtonWhat it does
■ InitPlace nodes on land, assign IDs, and populate routing tables. Re-clicking resets everything.
▶ Lookup TestRun Count random key lookups. Reports success rate, hop count (avg / p50 / p95 / max), and latency histograms.
↻ Churn TestSimulate node turnover over multiple intervals. Measures how routing health (success rate, hop count) degrades and recovers as nodes leave and join.
🌐 Slice WorldGenerates a network partitioned into Eastern and Western hemispheres, connected only through a single node in Hawaii. Tests how well each protocol discovers and uses a narrow bottleneck. Kademlia and G-DHT typically fail at ~52%; NX-10 succeeds 100% via iterative fallback and incoming-synapse reverse routing.
▶ DemoAnimate one lookup on the globe. The camera follows each hop; arcs shift yellow → orange → deep orange with distance.
⟳ Train NetworkRepeat lookup sessions in a loop for any protocol. All protocols improve with training: Kademlia and Geographic DHT update their k-buckets with fresher, more recently-seen peers as traffic flows through the network. Neuromorphic protocols additionally run a full learning pass each session (Markov annealing, hop-caching, triadic closure, LTP), producing larger and more sustained hop-count reductions. Press again to stop. Exports a CSV of all sessions.
⊕ Pub/SubFor older protocols (Kademlia, G-DHT, NX-6…NX-13): multi-group flat-broadcast test. A relay delivers to every participant via one lookup per subscriber. For NX-15 / NX-17: runs the live Membership Pub/Sub simulation instead — every node in a topic group subscribes through a real PubSubAdapter; the axonal tree grows dynamically; subscribers and axon nodes re-subscribe every tick; optional periodic churn kills a configurable % of non-publisher nodes; a bounded per-relay replay cache recovers messages subscribers missed during tree disruption. Reports immediate delivery %, cumulative delivery %, axon role count, max fan-out, tree depth, and publisher/subscriber K-overlap. Runs continuously; press again to stop.
⇔ Pair LearningEach node is assigned a fixed random target. Every session routes all pairs and records average hops. Measures how quickly the network memorises persistent routes.
🌡 HotspotTwo-phase load test. Highway: tracks which nodes relay the most traffic (Gini / Lorenz curve). Storage: queries content items with a Zipf distribution and measures query-load concentration.
▶▶ BenchmarkRuns every protocol (or the Fast subset) across many conditions: regional radii, source/dest-restricted, global, continent-crossing, and churn. No Init required.
Control Groups
GroupParameters
NetworkProtocol — routing algorithm. Nodes — network size (20 – 100K). K — bucket size (Kademlia default 20). α — parallel queries per round. Bits — ID bit-width (64 default). Delay — per-node processing latency added to every hop.
LookupCount — lookups per test run. Hot% — traffic concentration: 100% = uniform random; lower values focus traffic on a popular sub-network so neuromorphic protocols learn denser shortcuts.
ChurnRate — % of nodes replaced per interval. Int — number of churn cycles. L/Int — lookups measured per cycle.
FiltersRegion — restrict lookups to targets within a km radius. Src — only the top X% of nodes initiate lookups. Dest — only the top X% of nodes are valid targets.
Pub/SubGroup — participants per concordance group (relay + this many nodes, default 32). Cover % — percentage of network nodes that join at least one group (default 10%). More coverage creates more groups with overlapping routing paths.
NX-17 Pub/Sub (sidebar, shown when NX-17 is the active protocol)Max Subs / Min Subs — axon recruitment capacity hysteresis. Refresh (ms) — interval between axon self-subscribes. Sub TTL / Root Grace — aging timers. Churn per round (%) — fraction of alive non-publishers killed at each churn step during the live sim (accepts fractional values like 0.5, 0.3). Ticks per round — ticks between churn events. Refresh rounds / kill — refreshTick passes across all live nodes after each kill. K-overlap every N — diagnostic cadence (1 = every tick, larger = less overhead). Replay cache size — bounded per-relay ring of recent publishes used for subscribe-time replay (default 100).
HotspotQueries — number of lookups in the highway phase. Items — content items in the storage phase. Zipf — exponent controlling skew (0 = uniform, 1 = classic internet, 2 = extreme).
BenchWarmup — training sessions given to neuromorphic protocols before scoring (each session = 500 lookups). Protocols — multi-select of which protocols to benchmark. Tests — multi-select of which test conditions to run. Both support Select All / Deselect All. Settings are saved between sessions.
Protocols

Starred (★) protocols are Best in Class for their generation. NX-17 is the current State of the Art. Earlier NX versions are kept available for comparison and ablation work.

ProtocolKey idea
Kademlia ★Classical DHT baseline (2002). Random 64-bit IDs; XOR distance metric; O(log N) hops. K-bucket routing table with α parallel queries per round. Static — no learning. Performs identically regardless of where messages are destined.
G-DHT ★Geographic DHT — assigns IDs from physical location using an S2 Hilbert-curve prefix. Nearby nodes share high-order bits so routing paths follow geographic clusters. Combines stratified inter-cell allocation (40%), intra-cell locality (30%), and random global supplement (30%) for both latency and churn resilience. ~2× faster than Kademlia on local lookups; 100% success under 10% churn at 25 K nodes under the web-connection cap.
NX-17 ★★State of the Art. Neuromorphic routing layer (synaptome + AP scoring + LTP + annealing + churn-triggered temperature reheat + iterative fallback + two-tier highway) plus a distributed pub/sub membership protocol: publisher-prefix topic addressing (@XX/domain/event), routed subscribe with terminal-globality verification, root-only publish consumption, external-peer batch adoption on overflow, all-axon periodic re-subscribe for self-healing, and a bounded per-relay replay cache with subscriber-side lastSeenTs for missed-message recovery. 5–7× faster local routing than Kademlia; near-invariant under 25% churn; cumulative pub/sub delivery stays above 80% through 30% cumulative network churn without gossip, replication, or parent tracking.
NH-1Neuro-Homeostatic Protocol. A consolidation of NX-17's ~36 organically-grown rules into five fundamental operations — NAVIGATE, LEARN, FORGET, EXPLORE, STRUCTURE — expressed through a unified vitality model: vitality(syn) = weight × recency. A single _addByVitality() admission gate replaces stratified eviction, two-tier highway management, stratum floors, and synaptome floors. ~12 parameters replace NX-17's 44. Includes the full NX-17-style membership pub/sub layer (publisher-prefix topic IDs, single-root routed mode, batch-adoption, replay cache) on the same AxonManager backend, so NH-1 vs NX-17 is an apples-to-apples comparison on every benchmark including pubsubm+churn.

Earlier NX versions (research / comparison): N-1, N-15W, NX-1W, NX-2W, NX-3 — early neuromorphic designs and research protocols with configurable rule engines. NX-4 — the watershed: iterative fallback raises stress-test success from ~80% to 100%. NX-5/6 — stratified bootstrap, churn-resilient routing. NX-7/8/9 — three alternative dendritic pub/sub tree shapes (peel-off, balanced binary, geographic). NX-10 — former SOTA; routing-topology forwarding tree for pub/sub. NX-13 — NX-10 with all rules exposed for parameter ablation. NX-15 — first distributed pub/sub membership attempt (K=5 replication; superseded by NX-17). See Appendix A of the whitepaper for the full NX-1→NX-17 evolution, including the NX-16 masked-distance dead end.
